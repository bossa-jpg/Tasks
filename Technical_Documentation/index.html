Usage in Rack / Rails / Merb

To enable Sass in Rails version lower than 3, specify the environment.rbfollowing line in the file :

config.gem "sass"

For Rails 3, instead of the above, write to the Gemfile:

gem "sass"

To connect Sass in Merb version lower than 3, specify the config/dependencies.rbfollowing line in the file :

dependency "merb-haml"

To connect Sass in a Rack application, specify in the file config.ru:

require 'sass/plugin/rack' use Sass::Plugin::Rack

Sass itself does not work as a style sheet. It does not wrap dynamic content, in order for everything to work, you need to generate a CSS file immediately after updating the Sass file.

By default, files .sass and .scss are public/stylesheets/sass (can be changed with the option : template_location ). Then, when necessary, they are compiled into the appropriate CSS files in public/stylesheets. For example, it public/stylesheets/sass/main.scsswill be compiled to public/stylesheets/main.css.

Caching

By default, Sass caches compiled templates and fragments. This greatly speeds up the recompilation of large collections of Sass files, and works best if Sass templates are split into separate files that are imported into a single file.

Without the use of frameworks, Sass puts cached templates in a folder sass-cache. In Rails and Merb in a folder tmp/sass-cache. The folder can be changed using the option : cache_location . If you do not want to use the cache Sass, you can disable this feature if parameter : cache specify the value false.

Settings

Settings can be specified in the format Sass :: Plugin # options and written to environment.rbin Rails or config.ruin Rack ...

Sass::Plugin.options[:style] = :compact

... or setting up Merb::Plugin.config[:sass] to write init.rb to Merb ...

Merb::Plugin.config[:sass][:style] = :compact

... or by passing the hash to Sass :: Engine # initialize . All relevant options are available through the sass and scss labels on the command line.

Syntax Selection

If you work with Sass through the command line, then the syntax is determined through the file extension. However, the syntax can be chosen not only by the file extension. The default syntax is the indentation syntax, but you can change the syntax by applying a parameter --scss, and then the input file will be read as a file with scss syntax. In addition, you can use sass syntax commands with scss syntax and they will also be executed.

Encoding

When using Ruby 1.9 or later, Sass will determine the encoding of the file itself. To determine the encoding of the stylesheet, Sass uses CSS specifications , after determining Sass returns the encoding to Ruby for proper compilation. This means that the Unicode byte order is read first, followed by the directive @charset, then the Ruby string encoding. If none of them are strictly defined, then the document is assigned the default value - UTF-8.

To explicitly specify the style sheet encoding, use the directive @charset as in regular CSS. Add a construct @charset "name uncoding" at the beginning of the style sheet (without missing spaces and before any comments) and Sass will interpret this as the specified encoding. Please note that whatever encoding you use, it must be converted to Unicode.

Sass will always compile the resulting files in UTF-8 encoding. Such files will always have a record @charset if, and only in this case, the resulting file will contain non-ASCII content. In a compressed style in UTF-8 encoding, the bytes allocated for the directive @charset will be marked as used.
